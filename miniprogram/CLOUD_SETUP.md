# 联机房间(云开发)开通指南

德州 / 21 点 的「各自手机 · 建房联机」用的是 **微信云开发**(免服务器、免备案、国内直连)。
代码已全部写好,你只需要在微信开发者工具里点几下开通,再改一个环境 ID。

## 一次性开通(约 5 分钟)

1. **注册小程序 + 拿到 AppID**(个人主体即可,见 README.md)。游客模式无法用云开发,必须填真实 AppID。
2. 微信开发者工具导入本 `miniprogram/` 目录,填入你的 AppID。
3. 工具顶部点 **「云开发」** → 开通(选按量付费,**有免费额度**,聚会级用量基本 0 花费)→ 创建一个环境,记下左上角的 **环境 ID**(形如 `pour-xxxxx-yyyy`)。
4. 打开 `miniprogram/utils/cloudenv.js`,把 `REPLACE_WITH_YOUR_ENV_ID` 改成你的**环境 ID**。
5. 部署云函数:在工具左侧资源管理器里,右键 `cloudfunctions/room` 文件夹 → **「上传并部署:云端安装依赖」**。等它装好 `wx-server-sdk`。
6. 建数据库集合 + 权限(云开发控制台 → 数据库 → 新建集合):
   - 集合 **`rooms`** → 权限设为 **「所有用户可读,仅创建者可写」**(让房间里所有人都能实时 watch)。
   - 集合 **`roomState`** → 权限保持默认 **「仅创建者可读写」**(这是私密的底牌/牌堆,客户端读不到,只有云函数能碰)。

搞定。回到首页点 **「联机房间」** → 建房 → 把 4 位房号发群里 → 大家各自加入。

## 玩法
- 房主建房(选德州或 21 点)→ 生成 4 位房号 → 群里发号或点「邀请好友」分享。
- 各自进房,非房主点「准备」,房主点「开始」。
- **德州:每人手机只显示自己的两张底牌**,牌桌 / 池底 / 轮到谁全场实时同步;轮到你才可操作(弃/过/跟/加注 0.5–5 杯)。
- **21 点:明牌对局**,轮流坐庄,要牌/停牌。
- 摊牌自动比牌、算谁喝;房主点「下一局」继续。有人离开会自动散局回大厅。

## 说明 / 已知边界(v1)
- 这是**首个联机版本**,云端逻辑我在本地没法真机测,可能有 bug —— 你跑起来后把报错/异常截图发我,我来修。
- 断线:退出重进房间会自动重连(状态在云端)。
- 计费:`watch` 实时监听和云函数调用都在免费额度内,正常聚会用量放心。
- 其它游戏(点球 / 小姐牌 / 真心话)仍是**单机传手机**,不需要联机。
